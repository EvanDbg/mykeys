# MyKeys 后台管理界面需求文档

## 背景

当前密码管理操作全部通过企业微信聊天完成，存在不便：
- 批量操作困难
- 编辑复杂信息需多轮对话
- 无法直观查看和筛选数据

## 功能需求

### 1. 后台 API

提供 RESTful 接口用于密码管理：

| 接口 | 功能 |
|------|------|
| `GET /admin/api/secrets` | 获取密码列表 |
| `GET /admin/api/secrets/:id` | 获取单条详情 |
| `POST /admin/api/secrets` | 新增密码 |
| `PUT /admin/api/secrets/:id` | 修改密码 |
| `DELETE /admin/api/secrets/:id` | 删除密码 |

### 2. 认证机制

- 支持 Basic Auth 或 Bearer Token
- 通过环境变量配置管理员账户

### 3. 管理界面

Web 界面功能：
- 密码列表（表格形式）
- 搜索和过滤
- 新增/编辑对话框
- 删除确认
- 到期时间高亮提示

## 安全要求

- 生产环境强制 HTTPS
- 密码加密存储
- 请求频率限制

## 验收标准

- [ ] API 认证正常工作
- [ ] 能够完成密码的增删改查
- [ ] 界面响应良好，操作流畅
- [ ] 安全机制到位
